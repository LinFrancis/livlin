# Internacionalización (i18n) — LivLin

Sitio estático multilingüe generado en build. **No** requiere backend; funciona tal cual en GitHub Pages.

## Idiomas
| Código | URL | Carpeta de salida |
|--------|-----|-------------------|
| es (base) | `/` | raíz |
| en | `/en/` | `en/` |
| da | `/da/` | `da/` |
| zh-CN | `/zh/` | `zh/` |

## Estructura
- `src/` — **fuente única** de las páginas (en español). Edita aquí el HTML.
- `locales/<code>.json` — diccionarios de traducción `texto ES → traducción`.
- `locales/casos.<code>.json` — traducción de los casos de estudio (renderizados por `js/casos.js`).
- `i18n.config.mjs` — lista de idiomas y configuración.
- `build.mjs` — generador estático (sin dependencias).
- `js/i18n.js` — detección de idioma del navegador, persistencia y selector.

## Comandos
```bash
node build.mjs extract   # vuelca todos los textos a locales/_catalog.json
node build.mjs           # genera el sitio (raíz + /en /da /zh) + sitemap.xml + robots.txt
```
(o `npm run extract` / `npm run build`)

## Flujo de trabajo
1. Edita el contenido en `src/`.
2. `node build.mjs extract` para detectar textos nuevos.
3. Copia las claves nuevas de `locales/_catalog.json` a `en.json`, `da.json`, `zh-CN.json` y tradúcelas.
   *(Si una clave falta, se usa el español como respaldo — nada se rompe.)*
4. `node build.mjs`.
5. Commit de **todo** (incluida la salida generada en raíz, `/en`, `/da`, `/zh`).

## Agregar un idioma nuevo (ej. portugués `pt`)
1. Añade una entrada a `languages` en `i18n.config.mjs`:
   ```js
   { code: 'pt', dir: 'pt', label: 'Português', htmlLang: 'pt', ogLocale: 'pt_BR', hreflang: 'pt' }
   ```
2. Crea `locales/pt.json` y `locales/casos.pt.json`.
3. `node build.mjs`. Listo: URLs, selector, hreflang y sitemap se actualizan solos.

## SEO
Cada página generada incluye: `<html lang>` correcto, `canonical` propio, `hreflang` para los 4 idiomas + `x-default`, Open Graph (`og:url`, `og:locale` + alternates) y entrada en `sitemap.xml`.

## Notas
- Todos los archivos se escriben en **UTF-8 sin BOM**.
- El español es el idioma base: su texto se toma literal de `src/` (no necesita traducción).
- Las rutas de assets se reescriben a absolutas (`/css/...`) para que funcionen en cualquier nivel de carpeta.
